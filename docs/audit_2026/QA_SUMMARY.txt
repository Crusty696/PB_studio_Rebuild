KRITISCHSTE 5 BUGS
================================================================================

[B-010] TASK_MANAGER_INIT_TIMING — HOCH
  File: main.py:70
  Problem: TaskManager wird als Module-Level Code erstellt VOR QApplication.
           QApplication.instance() ist None → AttributeError
  Impact: CRASH beim Start
  Status: MUSS VOR RELEASE GEFIXT SEIN

[B-011] RACE_CONDITION_TASK_DICT — HOCH
  File: services/task_manager.py:290, 368
  Problem: _tasks dictionary wird von mehreren Threads modifiziert ohne Lock.
           Wenn Thread A modifiziert während Thread B iteriert: RuntimeError
  Impact: Crashes bei concurrent exports
  Status: MUSS VOR PRODUCTION GEFIXT SEIN

[B-006] MISSING_GPU_LOAD_LOCK — HOCH
  File: services/model_manager.py:425-439
  Problem: ensure_loaded() hat kein GPU_LOAD_LOCK.
           Mehrere Agenten können concurrent Models auf GPU laden.
  Impact: CUDA OOM, Segfault
  Status: MUSS GEFIXT SEIN WENN MEHRERE AGENTEN AKTIV

[B-002] MISSING_THREAD_TRACKING — HOCH
  File: main.py:807-809
  Problem: Bei Cross-Thread routing wird Thread nicht zu _active_threads.
           closeEvent() kann Thread nicht aufräumen.
  Impact: Thread leak
  Status: SOLLTE VOR RELEASE GEFIXT WERDEN

[B-012] SIGNAL_EMISSION_MISMATCH — HOCH
  File: main.py:776 + task_manager.py:232
  Problem: worker.progress wird 2x connected (zwei code paths).
           Progress updates feuern doppelt.
  Impact: Progress bar springt schneller, confusing UX
  Status: FIX EINFACH (1 connection entfernen)
